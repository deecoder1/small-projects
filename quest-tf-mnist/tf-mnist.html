def train(model, X_train, y_train, X_val, y_val, learning_rate=0.001, epochs=20, batch_size=32):
    model.compile(
        optimizer=tf.keras.optimizers.Adam(learning_rate=learning_rate),
        loss=tf.keras.losses.SparseCategoricalCrossentropy(from_logits=True),
        metrics=['accuracy']
    )
    
    # Train the model using the validation data for monitoring
    history = model.fit(
        X_train, y_train,
        epochs=epochs,
        batch_size=batch_size,
        validation_data=(X_val, y_val)
    )

    return history


def evaluate(model, X_test, y_test): 
    # After model selection, evaluate the final model on the test set
    test_loss, test_accuracy = model.evaluate(X_test, y_test)
    print(f"\n\nTest Loss: {test_loss:.4f}")
    print(f"Test Accuracy: {test_accuracy:.4f}")
    return test_loss, test_accuracy

def plot_training_statistics(history):
    _, axes = plt.subplots(2, 2, figsize=(10, 8))

    train_acc, train_loss = history.history['accuracy'], history.history['loss']
    val_acc, val_loss = history.history['val_accuracy'], history.history['val_loss']
    epochs = range(1, len(train_acc) + 1)

    # Training Accuracy
    axes[0, 0].plot(epochs, train_acc, 'bo-', label="Training Accuracy")
    axes[0, 0].set_title("Training Accuracy")
    axes[0, 0].set_xlabel("Epochs")
    axes[0, 0].set_ylabel("Accuracy")
    axes[0, 0].legend()

    # Validation Accuracy
    axes[0, 1].plot(epochs, val_acc, 'r*-', label="Validation Accuracy")
    axes[0, 1].set_title("Validation Accuracy")
    axes[0, 1].set_xlabel("Epochs")
    axes[0, 1].set_ylabel("Accuracy")
    axes[0, 1].legend()

    # Training Loss
    axes[1, 0].plot(epochs, train_loss, 'bo-', label="Training Loss")
    axes[1, 0].set_title("Training Loss")
    axes[1, 0].set_xlabel("Epochs")
    axes[1, 0].set_ylabel("Loss")
    axes[1, 0].legend()

    # Validation Loss
    axes[1, 1].plot(epochs, val_loss, 'r*-', label="Validation Loss")
    axes[1, 1].set_title("Validation Loss")
    axes[1, 1].set_xlabel("Epochs")
    axes[1, 1].set_ylabel("Loss")
    axes[1, 1].legend()

    plt.tight_layout()
    plt.show()

def train_evaluate_function(model):
    print("\n\n##### START TRAINING #####")
    history = train(
        model=model, 
        X_train=X_train/X_train.max(), # normalise to btwn 0-1
        y_train=y_train, 
        X_val=X_val/X_val.max(), # normalise to btwn 0-1
        y_val=y_val,
        epochs=10,
        batch_size=64
    )
    print("\n\n##### TRAINING COMPLETED #####")


    print("\n\n----- CHECK TRAINING STATISTICS -----")
    plot_training_statistics(history) # observe training and validation curves
    

    print("\n\n***** EVALUATE MODEL ON TEST DATASET *****")
    test_loss, test_accuracy = evaluate(
        model=history.model,
        X_test=X_test/X_test.max(),
        y_test=y_test
    ) # evaluate model on test set
    
    return history.model, test_loss, test_accuracy, history

Comparison of Results¶

In this experiment, three different models were trained on the MNIST dataset to evaluate their performance in digit classification. The models are:

  1. Simple FCN: A fully connected neural network with a single dense layer.
  2. Deep FCN: A deeper fully connected neural network with multiple dense layers.
  3. Deep FCN with Dropout: The same architecture as the second model, but with dropout layers added to regularize the network and reduce overfitting.

The aim was to compare their performance based on test accuracy and loss, as well as to examine the stability of the training curves during training.

Model Descriptions and Test Results¶

  1. Model 1: Simple FCN (Single Layer)

    • This model consists of a single fully connected (dense) layer, which is trained on the 28x28 grayscale MNIST images (flattened to 784 input features). The architecture is straightforward, with no additional regularization techniques, making it suitable for a baseline comparison.
    • Test Results for Model 1:
      • Test Loss: 0.1197
      • Test Accuracy: 0.9655

  2. Model 2: Deep FCN (Multiple Layers)

    • Model 2 is an enhanced version of the Simple FCN, containing multiple dense layers (256, 128, and 64 units). The additional layers increase the model's capacity to capture more complex patterns in the data. However, deeper networks are prone to overfitting, especially with small datasets like MNIST.
    • Test Results for Model 2:
      • Test Loss: 0.0937
      • Test Accuracy: 0.9754

  3. Model 3: Deep FCN with Dropout

    • Model 3 is similar to Model 2 but includes dropout layers after each dense layer. Dropout is a regularization technique that helps prevent overfitting by randomly "dropping" units during training, forcing the network to learn more robust features. This model is expected to show improved generalization over Model 2.
    • Test Results for Model 3:
      • Test Loss: 0.0745
      • Test Accuracy: 0.9800

Training Curves and Stability¶

  1. Model 1: Stable Training Curve

    • For Model 1, the training curves (accuracy and loss) are relatively stable. Since the model is simple with a single dense layer, it converges quickly without significant fluctuations. The loss steadily decreases, and the accuracy increases in a smooth manner. This is expected given the model's simplicity, which makes it less likely to overfit or underfit the training data.

  2. Model 2: Zigzagging Training Curve

    • Model 2 exhibits a more fluctuating (zigzagging) training curve, particularly in the early stages of training. This is due to the increased complexity of the model with multiple dense layers. The larger number of parameters in the network means that it is more prone to overfitting or underfitting, especially when training on a fixed-size dataset like MNIST. These fluctuations are common when the model struggles to find the optimal weights, leading to temporary increases in loss or instability in accuracy.

  3. Model 3: Stabilized Training Curve with Dropout

    • Model 3, which includes dropout after each dense layer, shows a stabilized training curve. The dropout mechanism prevents overfitting by forcing the network to learn more generalized features, which smooths out the zigzag behavior seen in Model 2. The dropout layers randomly disable certain neurons during training, helping the model avoid relying too heavily on any single feature or pattern. As a result, the training process becomes more stable and converges faster compared to Model 2.

Comparison and Analysis - Test Accuracy and Loss¶

  1. Model 1: The simple model achieves a test accuracy of 96.55% with a test loss of 0.1197. While these are decent results, the performance could be improved with a deeper model.

  2. Model 2: The deep model improves the accuracy to 97.54% and reduces the test loss to 0.0937, demonstrating the benefit of adding more layers. The increased number of parameters allows the model to capture more complex features from the data, leading to better performance.

  3. Model 3: Adding dropout to the deep model yields the best results, with a test accuracy of 98.00% and a test loss of 0.0745. This confirms that dropout helps to regularize the model, preventing overfitting while maintaining high accuracy.

Comparison and Analysis - Training Stability¶

  1. Model 1 shows a stable training curve, suggesting that the model does not experience overfitting or underfitting, but its relatively simple structure limits its ability to capture complex patterns.

  2. Model 2, with its deep architecture, experiences more fluctuations in the training curve, which can indicate overfitting or difficulty in convergence. This can happen due to the large number of parameters and the risk of overfitting to the training data.

  3. Model 3 benefits from dropout, which stabilizes the training process by preventing overfitting. This results in a smoother and more consistent training curve, ultimately improving generalization and model performance.

Conclusion¶

  • Model 1 is a good baseline model but is limited by its simplicity. While it performs well on MNIST, it fails to fully leverage the complexity of the data.
  • Model 2, with multiple dense layers, shows improved performance compared to Model 1. However, it struggles with training instability due to overfitting and the increased complexity of the model.
  • Model 3, which introduces dropout regularization, outperforms both Model 1 and Model 2 by achieving the highest test accuracy (98%) and the lowest test loss (0.0745). The addition of dropout not only improves performance but also stabilizes the training process, demonstrating the importance of regularization in deep learning models.

Overall, the results highlight the effectiveness of adding more layers and regularization techniques, such as dropout, to improve model performance and stability on complex tasks like digit classification with the MNIST dataset.
